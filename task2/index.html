A01:2021-Broken Access Control

Occurs when users can access resources or execute actions outside their intended permissions. Remediation involves implementing strict server-side access control checks, employing default-deny policies, and auditing API endpoints.

A02:2021-Cryptographic Failures

Formerly known as Sensitive Data Exposure. Focuses on failures related to cryptography, which often leads to exposed sensitive data. Mitigation includes encrypting all data in transit (TLS) and at rest (AES-256), and using strong hashing algorithms (bcrypt/Argon2).

A03:2021-Injection

SQL Injection, OS Command Injection, and LDAP Injection. Occur when untrusted user input is executed directly by an interpreter. Remediate by using parameterized queries (Prepared Statements), input validation, and proper escaping APIs.

A05:2021-Security Misconfiguration

Missing security headers, open ports, default accounts/passwords, or verbose error messages. Mitigated by automated hardening processes, removing unused services, disabling index listings, and configuring secure HTTP response headers.

Since browser scripts cannot perform actual TCP socket connections to arbitrary external domains, we have generated a robust companion python script `scanner.py` in your project directory. This script performs direct TCP port scans, inspects web service headers, and outputs vulnerability assessments.

# Step 1: Navigate to the project directory and execute the scanner script
python scanner.py localhost --ports 22,80,443,8080 --output report.html
# Step 2: The script scans for open ports, fetches banners, audits HTTP security,
# and generates a detailed report.html and report.json on disk.

Available CLI Arguments:

Argument Usage Description
target Positional Host name or IP address to target (e.g. 127.0.0.1, google.com).
--ports, -p Optional Comma-separated list of ports to scan. Default is common ports.
--output, -o Optional Custom filepath to export the HTML report.
--timeout, -t Optional Connection timeout in seconds. Default is 1.0s.
